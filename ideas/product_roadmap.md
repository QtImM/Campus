# 🚀 CampusCopy 产品进化路线图 (Product Roadmap)

这个文档记录了我们交流中产生的所有高价值想法（Ideas），旨在将 CampusCopy 从一个基础工具提升为具有核心技术壁垒的“高大上”应用。

---

## 🤖 AI 赋能模块 (AI Innovation)

### 🛡️ [进行中] AI 内容自动审核 (Trust & Safety AI)
- **目标**：实现语义级的自动化发帖审核，确保社区环境符合 App Store 的 UGC 政策。
- **技术栈**：OpenAI Moderation API / 自定义 LLM Prompt / 前端敏感词过滤优化。
- **亮点**：保护应用不被下架，展现风险控制与工程闭环能力。

“AI 内容自动审核”这个功能，属于典型的**“入门容易，做深非常有技术含量”**的项目。

为了方便你理解，我把它拆解成三个等级，你可以看看自己想挑战哪一个：

1. 初级难度：调用成熟 API（1-2天可完成）
这是最快的方式，也是目前 90% 的中型 App 的做法。

方案：使用 OpenAI 的 moderation 接口，或者百度/腾讯的审核 API。
逻辑：用户点击“发布” -> 前端将文字发给后端 -> 后端调 API -> 如果违规，返回错误提示。
技术点：API 调用、异步处理。
面试评价：证明你有“集成能力”和“产品合规意识”。
2. 中级难度：前后端联动 + 实时过滤（3-5天可完成）
在面试中，这才是面试官喜欢听的内容。

方案：
前端预过滤：建立一个本地的小型敏感词库（Aho-Corasick 算法），在用户输入时实时高亮或星号替代。
后端 AI 复审：发布时通过 AI 再次确认上下文语义（比如“你真牛”在不同语境下可能是夸奖也可能是阴阳怪气）。
技术点：高性能字符串匹配算法、React Native 的实时输入性能优化、Prompt Engineering（编写更聪明的审核提示词）。
面试评价：证明你懂“高性能算法”和“多层防御体系”。
3. 高级难度：多模态审核（图片 + 文字）（1周以上）
这是大厂（如字节、小红书）的核心技术栈。

方案：不仅审文字，还要审配图。利用大模型（如 GPT-4o-mini 或开源模型）识别图片里是否有违规内容、广告、甚至是不文明手势。
技术点：图片压缩转换流、多模态模型调用、并发审核队列。
面试评价：证明你具备处理深度学习/机器视觉应用的能力，属于溢出型人才。
对找工作的帮助（核心关键）：
在面试官眼里，这个功能的价值不在于“你会调个 API”，而在于你展示了：

风险控制意识：你懂 App Store 的规则（如果没有审核，App 会被直接下架），这说明你有商业化经验。
产品闭环能力：你不仅做了“点赞/发帖”，还考虑了“社区环境”，这说明你具备做千万级用户量产品的思维。
建议：
如果你想让项目显得“高大上”，我们可以搞一个**“三级安全过滤机制”**：

本地敏感词库（最快响应）
语义反垃圾 AI（理解语境）
恶意举报自动锁定（与我们之前的举报功能联动）
这样你在简历上就可以写：“主导设计了社区 Trust & Safety（信安）体系，利用多层过滤模型将违规内容拦截率提升至 XX%。” 

### 📸 AI 拍课表智能导入 (Schedule OCR)
- **目标**：拍照即读课表，自动精准提取课程名称、时间、地点，并一键导入日历。
- **技术栈**：深度学习 OCR (如 DeepSeek OCR / Donut) + 自然语言处理正则解析。
- **亮点**：极致的效率提升，涉及复杂的多模态数据转换。

### 🧠 校园知识智能问答 (Campus RAG)
- **目标**：基于《学生手册》和校规的私有知识库，回答关于选课、奖学金、行政事务的各种“疑难杂症”。
- **技术栈**：Vector Database (向量数据库) + RAG (检索增强生成) 架构。
- **亮点**：大语言模型在特定垂直领域的落地应用，具有极高的面试谈资。

### ✍️ AI 论文/作业润色助手
- **目标**：集成在发帖或评论区，帮助学生优化作业措辞、修正英文语法、快速生成长帖摘要。
- **技术栈**：LLM 文本生成与修复 API。
- **亮点**：增强社区内容的质量，提高用户粘性。

## 🗺️ 地图与生活增强 (Map & Lifestyle)

### 📍 智能导航连接线
- **目标**：在地图导航开启时，不仅显示标记，还绘制一条指引用户从当前位置到目的地的动态虚线。
- **状态**：已在导航 UX 计划中提及。

### 🍱 智能找吃的 (Find Food / Lifestyle Butler)
- **目标**：还原旧版本的筛选功能，并加入 AI 推荐算法。根据用户的专业（如“工科生时间紧”）或当前天气推荐最合适的餐厅。
- **状态**：待恢复并增强。

### 🔍 教室/自习室热度预报
- **目标**：实时显示各个地点的拥挤程度，帮助学生寻找安静的空位。
- **技术栈**：实时数据库 (Supabase Realtime) 或 众包数据统计。
